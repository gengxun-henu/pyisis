10 *_Title mocxtrack.h - MOC Cross Track Summing table maps
11 *_Args NONE

13 *_Descr This include file defines mapping of MGS MOC WA output pixels
14 * to hardware pixels. It is taken from the "Software Interface
15 * Specification Narrow Angle and Wide Angle Standard Data Products"
16 * publication, September 1999, formatted April 7, 2000, Appendix F.
17 * This document was provided by Malin Space Science Systems, Inc.,
18 * (MSSS) and specifies how on-board MOC WA varable summing relates
19 * to output pixel generation. These data are critical in proper
20 * radiometric calibration and geometric rectifications for these
21 * special WA cross track summing modes.
22 
23 * Note that it is very important to understand that the values
24 * in the starting and ending pixel are relative to the starting
25 * hardware detector pixel as specified in the FIRST_LINE_SAMPLE
26 * ISIS label keyword (EDIT_MODE_ID in MGS PDS labels). Thus,
27 * for example, the actual hardware pixel or pixels used to
28 * generate the first output pixel is:
29 *
30 * starting_hw_pixel = (FIRST_LINE_SAMPLE - 1) + table[0].starting_pixel
31 * ending_hw_pixel = (FIRST_LINE_SAMPLE - 1) + table[0].ending_pixel
32 *
33 * where "table" is either "mode27_table" or "mode13_table" depending
34 * on the value of CROSSTRACK_SUMMING in the ISIS label, 27 or 13,
35 * respectively. Generally, these images will always have the
36 * indicated number of samples per line for each mode.
37 */

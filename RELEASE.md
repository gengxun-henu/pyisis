# PyISIS v1.2.0

This is the first stable GitHub release of PyISIS, intended for Linux users who already have a working **USGS ISIS 9.0.0** environment.

## Support Scope

- Platform: Linux x86_64
- Python: CPython 3.12
- ISIS runtime: USGS ISIS 9.0.0
- Distribution: GitHub Releases, source builds, and Linux binary assets

## Release Assets

GitHub automatically provides source archives for each release:

- `Source code (zip)`
- `Source code (tar.gz)`

For this project, the recommended manually uploaded assets are:

- `isis_pybind-v1.0.0-linux-x86_64-cp312-isis9.0.0.tar.gz`
- `SHA256SUMS.txt`

The binary package should contain the complete `isis_pybind/` package directory, not just the `_isis_core*.so` shared library.

## Installation

### Option 1: Use the Release Binary Package

After extracting the archive, place the full `isis_pybind/` directory somewhere visible to the target Python environment, or temporarily set `PYTHONPATH`.

```bash
tar xzf isis_pybind-v1.0.0-linux-x86_64-cp312-isis9.0.0.tar.gz
export PYTHONPATH="$PWD/isis_pybind-v1.0.0-linux-x86_64-cp312-isis9.0.0${PYTHONPATH:+:$PYTHONPATH}"
python -c "import isis_pybind as ip; print(ip.__version__, hasattr(ip, 'Cube'))"
python tests/smoke_import.py
```

### Option 2: Build and Install from Source

```bash
export ISIS_PREFIX="$CONDA_PREFIX"
cmake -S . -B build -DCMAKE_BUILD_TYPE=Release \
  -DPython3_EXECUTABLE="$CONDA_PREFIX/bin/python" \
  -DISIS_PREFIX="$ISIS_PREFIX"
cmake --build build -j"$(nproc)"
cmake --install build
```

After installation, verify it with:

```bash
python -c "import isis_pybind as ip; print(ip.__version__, hasattr(ip, 'Cube'))"
python tests/smoke_import.py
```

## Runtime Requirements and Limitations

- The target machine must still provide a resolvable `libisis.so`.
- Qt-related shared libraries and camera/projection plugins are required.
- Many real-world workflows still require `ISISDATA` to be configured correctly.
- Support for Windows, macOS, or universal PyPI wheels is not currently provided.

## Integrity Check

After downloading the binary asset, you can verify it with:

```bash
sha256sum -c SHA256SUMS.txt
```

## Notes

If you only need the source code for this version, the source archives automatically generated by GitHub Releases are sufficient. In most cases, there is no need to upload an additional source `tar.gz` manually.

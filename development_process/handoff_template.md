# VS Code Copilot 长任务 handoff 模板

> 用途：在上下文窗口接近上限、任务尚未完成时，快速沉淀当前状态，并让下一个 Copilot 对话可以低成本恢复执行。
>
> 适用场景：pybind11 绑定、C++ 调试、测试补充、性能优化、跨多轮对话的长任务。

---

## 使用原则

- 只写 **事实、结论、约束、下一步**，不要写流水账。
- 优先写 **会影响下一个窗口继续执行** 的内容。
- 每完成一个自然里程碑就更新一次。
- 新窗口启动时，先读本文件中的“关键文件”，再动手修改。
- 如果有结构化进度文件（如 `pybind_progress_log.md` / `todo_pybind11.csv`），切换前一并同步。

## 推荐使用方式

### 什么时候更新

- 完成一个自然里程碑后更新一次。
- 准备切换主题前更新一次。
- 上下文开始“发涨”时，不要硬撑，先落盘再开新窗口。

### 怎么压缩

- 详细 handoff 放当前文件。
- 新窗口只复制“超短版摘要”。
- 详细事实写在 handoff，结构化进度写在 `pybind_progress_log.md` / `todo_pybind11.csv`。

### 新窗口启动建议

- 不要只说“继续上次工作”。
- 建议明确写：
  - 当前目标
  - 已完成内容
  - 关键文件
  - 下一步
  - 必须先重读的文件
  - 验证环境
